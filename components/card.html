§7 · Usage

Rules.

Do

  • Default to Flat. It fits most surfaces.
  • Use Outlined for nested cards (card inside card).
  • Use Subtle for asides and low-emphasis groupings.
  • Reserve Feature for the single attention card per view.
  • Opt in to interactivity explicitly via .card-interactive.
  • Keep header trailing slot to one item. More goes in footer.
  • Match radius and padding to the card size. Don't customize per-instance.
  • Wrap fully-clickable cards in a button or anchor.

Don't

  • Don't add drop-shadows. Ever. Surface contrast does the work.
  • Don't place two Feature cards on one screen.
  • Don't put a Hero button inside a Feature card. One anchor per view.
  • Don't override the 8px radius.
  • Don't use Feature variant for decoration. If it doesn't demand action, it isn't Feature.
  • Don't stack multiple primary buttons in a single card footer.
  • Don't attach onClick to a bare div. Use a semantic element or role.
  • Don't build one-off variants. New variants come back to this spec.
